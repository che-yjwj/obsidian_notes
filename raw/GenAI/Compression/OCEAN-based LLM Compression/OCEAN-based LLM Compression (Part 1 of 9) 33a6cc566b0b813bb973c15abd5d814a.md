    ## 제안: OCEAN — Outlier-Compensated Entropy-Aware Numerical coding
    
    ### 0) 목표 함수 관점(왜 이게 필요한가)
    
    일반 PTQ/QAT는 대체로 “MSE/퍼플렉시티”를 최소화하지만, 압축에서는 아래를 동시에 잡아야 합니다.
    
    • 정확도 손실
    
    • 비트레이트(코딩 길이)
    
    • 디코딩/런타임 비용
    
    즉, 실질적인 목적은[
    \min_{\hat{W}} \ \mathcal{L}_{task}(\hat{W}) + \lambda \cdot \mathcal{R}(\hat{W})
    ]
    인데, 여기서 (\mathcal{R})를 진짜로 줄이려면 **양자화 결과의 분포를 “엔트로피 코딩 친화적”으로 설계**해야 합니다.
    
    ## 1) Outlier를 “두 층”으로 분해한다: Base + Residual + (Sparse) Exception
    
    가중치 블록(예: row/col group, 64~256 elements)마다 다음처럼 분해합니다.
    
    [
    W = \underbrace{s \cdot Q}*{	ext{저비트 base}} \ + \ \underbrace{R}*{	ext{저엔트로피 residual}} \ + \ \underbrace{E}_{	ext{희소 예외(outlier)}}
    ]
    
    • Base (Q)
    
    • Residual (R)
    
    • Exception (E)
    
    여기서 포인트는 “outlier를 base 양자화에 억지로 포함시켜 전체 분포를 망치지 않는다”입니다.base의 symbol 분포가 잘 정리되면 Huffman/ANS에서 바로 이득이 나고, residual은 더 싸게 코딩됩니다.
    
    ## 2) 핵심 아이디어 A: Victim-pairing을 “코딩 친화”로 설계 (Outlier ↔ Victim의 목적 변경)
    
    기존 outlier-victim 계열은 주로 오차/정확도 관점에서 pair를 찾습니다. 여기서는 **pair를 “엔트로피 최소화” 관점으로 선택**합니다.
    
    ### 2.1 Pairing 목적함수
    
    블록 내에서 outlier 후보 (o)와 victim 후보 (v)를 골라, 다음 변환을 합니다:
    
    • base는 (Q)로 양자화
    
    • outlier의 잔차(혹은 보정)를 victim 쪽으로 “분배/상쇄”해서 residual의 분포를 중앙으로 모읍니다.
    
    pair 선택은 아래를 최소화:
    [
    \Delta H(Q) + \alpha \Delta H(R) + \beta \cdot 	ext{#exceptions}
    ]
    즉,
    
    • base symbol 분포가 더 뾰족해지도록(0, ±1 등 빈도↑)
    
    • residual이 0 근처에 몰리도록
    
    • exceptions 개수가 줄도록
    
    **정확도는 제약 조건**으로 넣습니다(블록별 허용 오차/민감도 기반).
    
    ### 2.2 왜 victim이 필요한가?
    
    outlier를 그냥 exception으로 빼면 인덱스 오버헤드가 생깁니다.victim과의 페어링으로 “exception을 줄이거나 residual로 흡수”하면,
    
    • index 오버헤드↓
    
    • residual은 작은 값 위주 → 엔트로피↓
    
    가 됩니다.
    
    ## 3) 핵심 아이디어 B: “Zero-Run을 만들기 위한” 엔트로피-유도 양자화 (Entropy-Induced Quantization)
    
    양자화 레벨을 MSE가 아니라 **symbol 빈도**가 좋아지게 설계합니다.
    
    ### 3.1 블록별 “쏠림 레벨 세팅”
    
    블록의 분산/민감도(예: Hessian diag 근사, 또는 GPTQ식 per-channel importance)로 **두 모드**를 둡니다.
    
    • Conservative mode (민감 블록)
    
    • Aggressive mode (둔감 블록)
    
    이때 중요한 건 “큰 step을 쓰되 outlier 때문에 분포가 꼬이지 않도록 1~2번의 보정 단계(Residual/Exception)를 같이 둔다”는 점입니다.
    
    ### 3.2 Residual을 ‘부호+크기’로 분해해서 코딩
    
    Residual (R)을 작은 정수로 제한:
    
    • (|R| \in {0,1,2,3}) 같은 형태로 만들고
    
    • 부호(sign)는 별도 비트플레인으로 분리
    
    • 크기는 Golomb/Rice 혹은 ANS로 코딩
    
    이 방식은 하드웨어에서 “작은 LUT/간단 누산”으로 복원이 가능하고, 비트스트림도 깔끔합니다.
    
    ## 4) 핵심 아이디어 C: “Permutation for Compression” — 실행과 무관한 재배열로 엔트로피 낮추기
    
    Weight는 저장 형태를 바꿔도 **행렬 곱 결과는 동일**하게 만들 수 있습니다(오프라인에서 입력/출력 채널 permute를 동반).
    
    • 예: Linear 레이어에서 output channel을 permute 하면, 다음 레이어의 input channel을 같은 permute로 보정하면 전체 기능 동일.
    
    이 자유도를 이용해,
    
    • outlier가 많이 나오는 채널을 한쪽으로 몰아 
    
    • 비슷한 통계를 가진 채널끼리 묶어 
    
    결과:
    
    • scale/zero-point 메타데이터 압축도 좋아지고
    
    • exception 인덱스도 연속화되어 RLE+ANS에 유리해집니다.
    
    ## 5) 비트스트림 포맷(하드웨어 친화)
    
    레이어별로 다음 “3-스트림” 구조를 추천합니다.
    
    1. Base stream (Q)
    
    • 블록 단위로 INT4/INT3 packed
    
    • 추가로 “심볼 재매핑(0, ±1 등) + Huffman/ANS” 적용 가능
    
    • 랜덤 액세스 위해 블록별 오프셋 테이블
    
    1. Residual stream (R)
    
    • 블록별 residual 존재 여부 bitmask
    
    • residual 값은 small alphabet(0..k) → ANS/Huffman
    
    • sign은 별도 bitplane(압축 가능)
    
    1. Exception stream (E)
    
    • 블록별 exception count
    
    • index는 delta coding + RLE
    
    • value는 INT8/FP16 중 택1 (outlier는 몇 개 안 되니 FP16도 허용 가능)
    
    이 구조는 NPU에서
    
    • base는 그냥 디패킹해서 MAC에 공급
    
    • residual/exception은 “옵션 경로”로 누산에 더함
    
    로 구현 가능합니다. (성능이 중요한 경우 residual/exception을 pre-decode해서 SRAM에 풀어두는 옵션도 가능)

    ## 왜 dead-zone이 압축에 바로 이득인가
    
    Dead-zone은 보통 다음 형태입니다(대칭 quant 가정):
    
    • 일반 uniform: 0에 가장 가까운 구간 폭 = (\Delta)
    
    • dead-zone: 0 구간 폭 = (k\Delta) (보통 (k\in[1.5,3]))
    
    그러면 결과적으로
    
    • 0 심볼 빈도 급증
    
    • 작은 값들이 0으로 흡수 → 
    
    • 블록별로 “run-length(연속 0)”도 늘어남 → RLE+ANS 조합에 유리
    
    즉, 당신이 제안한 **OCEAN(Outlier-aware + entropy-aware)**의 “base stream”을 엔트로피 관점에서 정리하는 가장 쉬운 방법이 dead-zone입니다.

    dead-zone은 작은 weight를 0으로 더 많이 보내서
    
    • attention/FFN의 
    
    • 특히 
    
    그래서 “모든 블록에 k=2 고정” 같은 방식은 위험합니다.
    
    ## 추천: “선택적 dead-zone + outlier/residual과 결합”
    
    ### 1) 블록별 on/off 또는 (k)를 적응적으로
    
    블록 (B)에 대해 아래 지표로 dead-zone 강도를 결정:
    
    • kurtosis / heavy-tail 지표
    
    • importance(민감도)
    
    • 민감 블록: (k \approx 1.0\sim1.5)
    
    • 둔감 블록: (k \approx 2.0\sim3.0)
    
    실무적으로는 **블록별로 2비트 정도의 “DZ mode” 메타데이터**만 붙여도 충분합니다(00:off, 01:k=1.5, 10:k=2, 11:k=3).
    
    ### 2) dead-zone은 base에만 적용, residual은 “작은 알파벳”로 제한
    
    구조는 이렇게 가면 깔끔합니다.
    
    • Base (Q): dead-zone 적용 → 0 빈도↑
    
    • Residual (R): (|R|\in{0,1,2,3}) 같은 작은 정수 제한 → ANS로 매우 싸게 코딩
    
    • Exception (E): dead-zone으로 못 담는 것만 처리
    
    이 조합이 “0을 늘려 압축”하면서도 “정확도는 residual로 복구”하는 가장 안전한 형태입니다.
    
    ### 3) outlier-aware pairing과의 시너지
    
    dead-zone이 커지면 victim 후보(작은 값)가 0으로 더 쉽게 빨려 들어가 **victim 풀이 줄어들 수** 있습니다.그래서 pairing 단계에서:
    
    • dead-zone을 먼저 적용해 base를 만든 뒤
    
    • 남은 outlier에 대해 “코딩 비용 + 정확도” 기준으로 victim을 고르는 순서가 더 안정적입니다.

    ## 1) “정확도 예민 구간”이 생기는 이유 (dead-zone이 위험해지는 조건)
    
    dead-zone은 작은 weight를 0으로 몰아 압축엔 유리하지만, 어떤 블록은 “작은 값들이 실제로 기능적으로 중요”합니다.
    
    블록이 예민해지는 대표 조건:
    
    • **입력 활성 (x)**가 큰 채널(또는 토큰/헤드)과 곱해져서 작은 (w)도 출력에 크게 기여
    
    • **출력 프로젝션류(O_proj, FFN down)**처럼 다음 단계로 “정보를 다시 섞는” 경로에서 작은 성분 손실이 누적
    
    • 초기 레이어
    
    하지만 이건 “대략적” 설명이고, 실제 결정은 아래처럼 **블록 단위 지표로 내리면 명확해집니다.**
    
    ## 2) 블록(32 weights)에서 “정밀 표현 vs dead-zone” 결정 기준: 엔트로피-왜곡 최적화
    
    당신이 짚은 케이스가 핵심입니다:
    
    > 
    
    32개 그룹이 outlier 없이 **매우 작은 값들**만 있을 때scale/zero-point로 정밀하게 표현할지, dead-zone으로 날릴지
    
    이 결정은 결국 아래의 **Rate–Distortion** 비교로 “명확화”할 수 있습니다.
    
    ### 2.1 블록별 후보 모드 정의
    
    블록 (B)에 대해 두 가지(혹은 3가지) 모드를 비교합니다.
    
    **Mode A: Fine Quant (정밀 양자화, dead-zone 약/없음)**
    
    • 작은 (\Delta_A)로 quant → 왜곡 (D_A) 작음
    
    • 하지만 심볼이 더 다양해져 엔트로피 (R_A) 증가(코딩 비트↑)
    
    **Mode B: Dead-zone Aggressive (dead-zone 큼)**
    
    • 더 많은 0 → 엔트로피 (R_B) 감소(코딩 비트↓)
    
    • 대신 왜곡 (D_B) 증가
    
    (선택) **Mode C: Dead-zone + Small residual**
    
    • base는 DZ로 0 몰기 → (R)↓
    
    • residual을 작은 알파벳(±1..±k)로 보정 → (D)를 다시 낮춤, 비트는 조금만 증가
    
    이게 실무적으로 가장 “안정적”인 중간해입니다.
    
    ### 2.2 의사결정: “비트 절감 vs 출력 손실”
    
    블록별로 다음을 계산해 비교하면 됩니다.
    
    [
    J = D + \lambda R
    ]
    
    • (D): 출력에 대한 기대 왜곡(정확도 손실 proxy)
    
    • (R): 엔트로피 코딩 후 기대 비트 수(혹은 추정 엔트로피)
    
    • (\lambda): “비트 1개가 정확도 손실 얼마와 맞먹는지”를 조절
    
    결정 규칙은 단순히:
    
    • (J_B < J_A)면 dead-zone(또는 C) 선택
    
    • 아니면 정밀 양자화(A) 선택
    
    여기서 “명확한 기준”의 관건은 **D와 R을 블록에서 어떻게 계산하느냐**입니다.
    
    ## 3) 엔트로피 코딩 관점에서의 R(비트) 추정: 블록의 “심볼 분포”로 계산
    
    블록이 32개면, mode별로 실제 quant를 한 번 해보고 심볼 히스토그램을 만들어서 **ANS/Huffman의 기대 비트**를 근사할 수 있습니다.
    
    ### 3.1 Base 스트림 비트(가장 중요)
    
    mode별로 양자화 결과 심볼 (q_i)의 분포 (p(s))를 얻고,
    [
    R_{	ext{base}} \approx 32 \cdot H(p) \quad (H=-\sum p\log_2 p)
    ]
    
    • dead-zone이 잘 먹히면 (p(0))가 커지고 (H)가 크게 떨어집니다.
    
    • 반대로 정밀 모드에서 심볼이 고르게 퍼지면 (H)가 증가합니다.
    
    ### 3.2 Residual/Exception 비트
    
    • residual을 쓰는 모드(C)라면 residual 알파벳 분포로 동일하게 (R_{	ext{res}}) 추정
    
    • exception은 인덱스+값 비용이 있으니, 블록당 outlier 개수 (k)에 대해
    
    **핵심:**블록 크기 32에서는 “정확한 ANS 상태 머신까지” 넣지 않아도, **엔트로피(H) + 고정 오버헤드**만으로도 의사결정이 매우 잘 됩니다.
    
    ## 4) 정확도 관점의 D(왜곡) 추정: “입력 에너지 가중 MSE”가 제일 실용적
    
    LLM weight는 **출력 오차**가 중요하므로, 블록 (B)가 속한 output 채널 (j)에 대해:
    
    [
    D \approx \mathbb{E}[x^2] \cdot \sum_{i\in B} (\Delta w_i)^2
    ]
    
    • (\Delta w_i = w_i - \hat{w}_i)
    
    • (\mathbb{E}[x^2])는 해당 레이어 입력의 2차 모멘트(캘리브레이션 데이터로 측정)
    
    • 더 정교하게는 GPTQ류처럼 Hessian diag (h_i)를 써서
    
    이렇게 하면 “작은 값이지만 입력이 큰 채널”에서 dead-zone을 자동으로 억제하게 됩니다.즉, 사용자가 걱정한 **“작은 값이지만 정밀 표현해야 하는 블록”**을 지표가 잡아냅니다.
    
    ## 5) 당신이 제시한 특수 케이스(32개가 전부 매우 작은 값): 명확한 판단 예시

    "From Entropy to Epiplexity"
    
    인공지능, 특히 거대언어모델(LLM)과 알파제로(AlphaZero)가 보여주는 비약적인 성취는 우리가 굳게 믿어왔던 고전 정보이론의 기반을 송두리째 뒤흔든다.
    
    체스의 단순한 규칙만 입력받은 시스템이 어떻게 스스로 초인적인 전략을 창발해낼 수 있는가? 단순히 다음 토큰을 예측하도록 훈련받은 모델이 어떻게 고차원적인 추론과 일반화 능력을 획득하는가?
    
    이러한 현상들은 우리에게 "과연 정보란 무엇인가?"라는 질문을 원점에서 다시 던지게 만든다.
    
    클로드 섀넌의 엔트로피와 콜모고로프로의 복잡도로 대표되는 고전적 관점에서 정보의 양은 곧 불확실성, 즉 무작위성(Randomness)과 동치였다.
    
    이 정의에 따르면 화이트 노이즈처럼 패턴이 전무하여 압축 불가능한 데이터가 가장 많은 정보를 담고 있는 셈이 되며, 결정론적 과정은 새로운 정보를 생성할 수 없다고 간주된다. 그러나 딥러닝의 경험적 현실은 이 직관과 정면으로 충돌한다.
    
    자연어, 이미지, 소스 코드처럼 풍부한 구조(Structure)를 가진 데이터는 학습 가치가 매우 높지만, 기존 이론상 정보량이 최대인 난수나 해시 값은 지능 시스템에 아무런 통찰도 제공하지 못하기 때문이다. 즉, 기존 이론은 ‘학습 가능한 의미 있는 데이터’와 ‘단순한 무질서’를 명확히 구분하지 못하는 치명적인 한계를 드러낸다.
    
    논문 "From Entropy to Epiplexity"는 이 괴리가 정보의 정의에서 ‘관찰자의 계산 자원 제약(Computational Constraints)’을 소거했기 때문에 발생한다고 지적한다.
    
    정보는 데이터 속에 절대적으로 존재하는 유령이 아니라, 제한된 계산 능력을 가진 관찰자가 그 안에서 얼마나 구조를 해독해낼 수 있느냐에 따라 결정되는 상대적 실체라는 것이다. 저자들은 이 관점을 정립하기 위해 에피플렉시티(Epiplexity)와 시간 제한 엔트로피(Time-bounded Entropy)라는 새로운 개념을 도입한다.
    
    여기서 에피플렉시티란, 제한된 계산 자원 내에서 모델이 데이터를 압축하며 파라미터에 내재화한 ‘구조적 정보’를 뜻한다. 이는 모델이 계산을 투입하여 해소해 낸 불확실성의 양이다. 반면 시간 제한 엔트로피는 해당 모델이 주어진 시간 내에 설명하지 못하고 남겨둔 잔여 불확실성으로, 학습 과정의 손실 함수(Loss)와 직결된다. 이 프레임워크에서 정보는 더 이상 데이터 자체의 속성이 아니라, 관찰자가 자신의 계산 예산을 태워 데이터로부터 추출해 낸 ‘구조의 총량’으로 재정의된다.
    
    이 새로운 렌즈를 끼우면 오래된 정보의 역설들이 비로소 해상도를 갖기 시작한다. 먼저, 결정론적 과정은 정보를 생성할 수 없다는 통념이 깨진다. 콘웨이의 ‘생명 게임(Game of Life)’이나 세포 오토마타는 규칙 자체는 단순하지만, 계산적으로 기약 불가능(Computationally Irreducible)한 과정을 거치며 관찰자에게 글라이더와 같은 고수준의 구조적 정보를 창발해낸다.
    
    물리적 엔트로피의 증가 없이도, 관찰자의 에피플렉시티는 증가할 수 있는 것이다. 또한, 데이터의 순서가 정보량과 무관하다는 가정 역시 무너진다. 암호학적 예제나 체스 기보 학습이 증명하듯, 적절한 커리큘럼은 시간 제약이 있는 학습자에게 잠재된 구조를 더 효율적으로 노출시킨다. 이는 데이터의 순서가 모델이 추출할 수 있는 구조적 정보의 상한선을 결정짓는다는 것을 의미한다.
    
    나아가 이는 가능도(Likelihood) 기반 학습이 단순한 통계적 분포 맞추기에 불과하다는 비판을 반박하는 강력한 논거가 된다. 소위 ‘Sutskever의 추리 미스터리’가 시사하듯, 다음 토큰을 완벽하게 예측하기 위해서는 결국 데이터를 생성한 잠재적 메커니즘을 이해해야만 한다. 즉, 가능도를 최적화하는 과정은 단순한 암기가 아니라, 데이터 기저의 알고리즘을 역공학하여 구조를 학습하는 과정과 같다고 볼 수 있다.
    
    이 이론은 알파제로가 인간의 기보 없이도 초월적인 전략을 구사하고, LLM이 텍스트 나열에서 문법과 논리라는 재사용 가능한 인지적 서브프로그램을 획득하는 현상을 명쾌하게 설명한다. 그들은 데이터의 무작위성을 학습한 것이 아니다.
    
    알파제로는 방대한 탐색 계산을 통해 상태 공간에 내재된 구조를 추출했고, LLM은 텍스트의 순서 속에 숨겨진 생성 규칙을 파라미터로 압축해 낸 것이다. 핵심은 데이터의 절대적 엔트로피가 아니라, 모델이 제한된 계산 예산 안에서 포착해 낼 수 있는 에피플렉시티의 양이다.
    
    결국 우리는 정보에 대한 정의를 다시 써야 한다. 지능형 시스템에게 중요한 질문은 "데이터가 얼마나 랜덤한가"가 아니라, "얼마나 계산적으로 구조화되어 있으며 재사용 가능한가"이다.
    
    무작위성은 정보의 극대화가 아니라 오히려 계산적 학습 불가능성을 의미할 뿐이다.
    
    엔트로피에서 에피플렉시티로의 패러다임 전환은 현대 인공지능의 성공을 이론적으로 정당화할 뿐만 아니라, 향후 데이터 설계와 모델 평가의 기준이 '무질서의 측정'에서 '구조의 발견'으로 옮겨가야 함을 시사하는 중요한 이정표가 될 것이다.

    1. dead-zone 적용 시 residual이 체계적 패턴을 가짐
    
    반대로:
    
    • 독립적
    
    • 입력 의존성 낮음
    
    • 분산 작고 패턴 없음
    
    → **Epiplexity 낮음 → 제거 대상**
    
    👉 이게 바로**“작은 값이지만 날리면 안 되는 블록”을 구분하는 기준**입니다.

    ## 2. LLM 관점에서 “작은 값 = 화이트 노이즈?” ❌

    • 랜덤하다

    • 구조가 없다
    
    • 버려도 된다
    
    즉,
    
    > 
    
    **작은 값 ≠ 화이트 노이즈**
    
    ## 3. 작은 값이 “진짜 화이트 노이즈”인 경우 (버려도 되는 경우)
    
    아래 조건을 **대부분 만족**하면, 그 작은 값들은
    Epiplexity 관점에서도 **버려도 되는 잔여(Time-bounded entropy)**에 가깝습니다.
    
    ### 조건 A: 통계적 독립성
    
    • 블록 내에서 서로 상관이 없음
    
    • 패턴 없이 부호가 랜덤하게 바뀜
    
    ### 조건 B: 입력 의존성 없음
    
    • activation-weighted sensitivity가 매우 작음
    
    ### 조건 C: 반복 사용되지 않음
    
    • 특정 토큰/헤드/컨텍스트에서만 간헐적으로 등장
    
    • 여러 입력에서 일관된 역할을 하지 않음
    
    ### 조건 D: 제거 후 residual이 구조를 만들지 않음
    
    • dead-zone 적용 시 residual이 거의 없음
    
    • 혹은 residual이 또다시 랜덤
    
    👉 이 경우:
    
    • 엔트로피는 높을 수 있지만
    
    • Epiplexity는 낮음
    
    • dead-zone / pruning 정당화
    
    ## 4. 작은 값이지만 “절대 화이트 노이즈가 아닌” 경우 (보존 대상)

    ### Case 1: 작은 값이지만 **입력 에너지가 큼**
    
    [
    y = \sum w_i x_i
    ]

    • (x_i)가 항상 큼

    • 스케일 조정자
    
    • 정밀한 균형 파라미터
    
    • 구조적 역할 수행
    
    ➡️ **Epiplexity 높음**
    
    ### Case 2: 작은 값들이 **집단적으로 패턴을 이룸**
    
    • 개별 값은 작지만
    
    • 블록 전체가 저차원 subspace / 특정 방향성 유지
    
    • dead-zone 후 residual이 
    
    이건:
    
    • 노이즈 ❌
    
    • 저해상도로 표현된 구조
    
    ➡️ residual로 보존해야 할 정보
    
    ### Case 3: 반복적으로 재사용되는 “서브프로그램”
    
    LLM 내부에는:
    
    • 문법적 보정
    
    • attention re-weight
    
    • logits 미세 조절
    
    같은 **인지적 마이크로 서브루틴**이 있습니다.
    
    이때:
    
    • 큰 weight는 coarse structure
    
    • 작은 weight는 fine structure
    
    ➡️ 값은 작아도 **재사용성 높음 = Epiplexity 큼**

    📌 의미:
    
    > 
    
    **LLM의 Epiplexity 중 ‘핵심 골격’은INT4에서도 표현 가능하지만,미세 구조는 별도 보호가 필요**
    
    → **INT4는 Epiplexity-critical bound**
    
    ### (C) INT3 / INT2 (≤ 2 bits)
    
    **Epiplexity의 구조적 하한이 드러나는 영역**
    
    • 일부 레이어/헤드는 가능
    
    • 전체 모델에 전면 적용은 현재 구조적으로 어려움
    
    • 이유는 단순 정밀도 부족이 아니라:
    
    • sign
    
    • 상대 크기
    
    • 조합 관계
    
    를 동시에 표현 불가
    
    📌 의미:
    
    > 
    
    **INT3 이하부터는‘표현 부족’이 아니라‘구조 표현 능력의 붕괴’가 발생**
    
    → **INT3 ≈ Epiplexity lower bound (현재 아키텍처 기준)**
    
    ## 4. 구조 단위별 “최소 비트 수” 직관적 정리
    
    아주 중요한 표입니다.
    
    | 구조 단위 | Epiplexity 성격 | 최소 비트(경험적) |
    | --- | --- | --- |
    | Attention routing | 강한 구조 | 3~4 bits |
    | FFN coarse mapping | 저차 구조 | 2~3 bits |
    | Logit fine tuning | 미세 구조 | 4~6 bits |
    | Outlier weights | 구조적 스파이크 | 예외 처리 필요 |
    | Residual correction | 구조 보정 | 2~3 bits |
    
    👉 그래서 **mixed precision이 본질적으로 필요**합니다.
    
    ## 5. Epiplexity 관점에서의 “진짜 하한” 정의
    
    이 질문에 대한 가장 정확한 정의는 이것입니다.
    
    > 
    
    **Epiplexity의 최소 표현 비트 수란“모델이 학습한 구조를계산적으로 재현 가능한 형태로더 이상 분해할 수 없는 최소 표현 복잡도”이다.**
    
    이것은:
    
    • weight 비트 수
    
    • activation 비트 수
    
    • 연산 구조
    
    • residual path
    
    의 **조합적 결과**입니다.
    
    즉,
    
    > 
    
    **INT4 하나로 끝나는 문제가 아니라‘INT4 + residual + outlier path’의 총합이 하한**
    
    ## 6. 그래서 지금 시점에서의 가장 정확한 답
    
    > 
    
    **LLM Epiplexity의 최소 표현 비트 수는단일 정수가 아니라,구조 분해 이후의 혼합 표현으로서평균적으로 2~4 bits/weight 수준에 수렴한다.**
    
    • 단, 이건:
    
    • outlier 분리
    
    • dead-zone
    
    • residual 보정
    
    • 구조-aware scaling
    
    을 전제로 한 값입니다.
    
    ## 7. 가장 중요한 오해 하나를 바로잡자
    
    ❌ “INT4가 되니까 정보량이 4비트다”❌ “INT8이 되니까 정보량이 8비트다”
    
    ✅ **정보(Epiplexity)는 비트 수에 직접 대응하지 않는다.**비트는 **구조를 담는 용기**일 뿐입니다.
    
    ## 8. 한 문장으로 최종 답
    
    > 
    
    **Epiplexity의 최소 표현 비트 수는모델 전체에 대해 하나로 정의되지 않으며,현재의 Transformer LLM에서는구조 분해를 전제로 할 때평균적으로 2~4 bits/weight가‘의미 있는 하한’으로 관측된다.INT8은 안전 영역이고,INT4는 구조 보호가 필요한 경계이며,INT3 이하는 구조적 재설계 없이는 어렵다.**
    
    이 답이 지금까지 당신이 관찰한 **INT8 무손실 / INT4 난이도 급증** 현상을가장 정확하게 설명합니다.
